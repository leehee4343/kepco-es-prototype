핵심 판단 기준은 원본 보존, 지원 파일 형식, 워터마크 적용 시점, 다운로드 파일의 보안성입니다.

워터마크·타임스탬프 적용 업무 프로토타입

파일 업로드 이후 문서 열람·다운로드 과정에서 기관 로고 워터마크와 업로드 시각 타임스탬프를 적용하는 두 가지 방식, 즉 사이냅 문서뷰어 방식과 오즈리포트 방식의 업무 흐름과 장단점을 비교합니다.

📑
리포팅툴 방식업로드 시점에 보안 정보가 포함된 PDF 생성
🛡️
공통 목적워터마크와 타임스탬프를 통한 문서 유통 통제
⚠️
핵심 차이원본 유지 중심인지, 보안 적용 파일 생성 중심인지
👁️

1. 사이냅 문서뷰어 도입 방식

업무 전개 절차

파일 업로드 후 목록 화면에서 바로보기와 다운로드 기능을 분리하여 제공합니다.

1
파일 업로드사용자가 원본 파일을 시스템에 등록합니다.
2
목록 화면 버튼 제공업로드된 파일 옆에 ‘바로보기’와 ‘다운로드’ 버튼을 제공합니다.
3
바로보기 클릭사이냅 문서뷰어가 열리고, 뷰어 내에서 기관 로고 워터마크와 업로드 시각 타임스탬프를 표시합니다.
4
뷰어 내 다운로드워터마크와 타임스탬프가 적용된 PDF 파일을 다운로드합니다.
5
일반 다운로드문서뷰어 외부에서 다운로드할 경우 워터마크와 타임스탬프가 적용되지 않은 원본 파일을 다운로드합니다.
장점: 원본 품질 유지 장점: 다양한 파일 형식 대응 단점: 원본 직접 적용 불가 주의: 일반 다운로드는 원본 제공
📊

2. 오즈리포트 도입 방식

업무 전개 절차

업로드 단계에서 PDF 파일을 리포트 레이아웃으로 변환하고, 워터마크와 타임스탬프가 포함된 신규 PDF를 생성합니다.

1
PDF 파일 업로드오즈리포트에서 인식 가능한 PDF 형식으로 파일을 등록합니다.
2
빈 리포트 레이아웃 생성오즈리포트의 빈 레이아웃 안에 PDF 내용을 자동 삽입합니다.
3
워터마크·타임스탬프 적용파일 업로드 시점에 기관 로고와 업로드 시간을 PDF에 반영합니다.
4
보안 적용 PDF 생성원본을 기반으로 워터마크와 타임스탬프가 포함된 PDF를 새로 생성합니다.
5
다운로드 후 확인별도 문서뷰어 없이 다운로드한 PDF 파일로 문서를 확인합니다.
장점: 보안 적용 PDF 생성 장점: 별도 뷰어 불필요 단점: PDF만 가능 주의: 화질 저하·업로드 지연 가능
⚖️

3. 두 방식 비교

비교 항목사이냅 문서뷰어오즈리포트
기본 개념문서를 열람할 때 보안 정보를 표시하는 조회 중심 방식업로드 시 보안 정보가 포함된 PDF를 생성하는 파일 생성 중심 방식
지원 파일HWP, DOCX, XLSX, PDF 등 다양한 문서 형식 대응 가능PDF 파일만 업로드 가능
적용 시점문서뷰어에서 바로보기 또는 뷰어 내 다운로드 시 적용파일 업로드 및 PDF 생성 시점에 적용
원본 보존원본 파일을 변경하지 않음원본 기반의 신규 PDF 생성
다운로드 결과뷰어 내 다운로드는 보안 PDF, 일반 다운로드는 원본 파일다운로드 파일 자체가 보안 적용 PDF
품질 영향원본 품질 훼손 없음PDF 재생성 과정에서 화질 저하 가능
처리 속도업로드 속도 부담이 상대적으로 적음변환·생성 과정으로 업로드 시간이 길어질 수 있음
✅

4. 적용 방향 판단

사이냅 문서뷰어가 적합한 경우

다양한 원본 문서 형식을 유지해야 하고, 원본 품질 훼손 없이 열람 과정에서 워터마크와 타임스탬프를 제공하는 것이 중요한 경우 적합합니다.

오즈리포트가 적합한 경우

사용자가 다운로드하는 파일 자체에 워터마크와 타임스탬프가 반드시 포함되어야 하며, PDF 중심의 문서 유통 체계가 가능한 경우 적합합니다.

핵심 메시지

문서뷰어 방식은 원본 보존과 다중 파일 형식 지원에 강점이 있습니다.
리포팅툴 방식은 보안 적용 PDF를 직접 생성할 수 있다는 점이 강점입니다.
보안 요구사항이 “열람 통제”인지 “배포 파일 자체 통제”인지에 따라 선택 기준이 달라집니다.

검토 필요 사항

일반 다운로드에서도 워터마크 적용이 필수인지 확인해야 합니다.
업로드 대상 파일 형식이 PDF로 제한 가능한지 검토해야 합니다.
PDF 재생성 시 화질 저하와 처리 시간 증가를 수용할 수 있는지 확인해야 합니다.
